# Romashsq Autoservice Bot — Technical Specification

**Version:** 2.0
**Stack:** Node.js 22 · grammY · MongoDB Atlas · Google Sheets API · Render.com
**Repo:** https://github.com/Romashsq/romashsqAUTO
**Business:** Romashsq Autoservice, вул. Рішельєвська 43, Одеса. Пн–Пт 09:00–18:00

---

## 1. Architecture Overview

```
Telegram ──POST──▶ Render (HTTPS webhook) ──▶ grammY Bot
                                                    │
                        ┌───────────────────────────┤
                        │                           │
                   MongoDB Atlas              Google Sheets
                  (primary store)          (admin CRM mirror)
```

- **MongoDB** — єдине джерело правди для всіх операцій бота
- **Google Sheets** — дзеркало для адміна (перегляд через браузер, без логіки)
- **Session** — стан wizard'у зберігається в MongoDB (не в пам'яті!)

## 2. Critical Bugs (виправити до релізу)

| # | Баг | Причина | Рішення |
|---|-----|---------|---------|
| 1 | ❌ Кнопка "Підтвердити" не реагує | State в пам'яті (Map) — стирається при рестарті Render | Перенести session у MongoDB |
| 2 | ❌ Нагадування не надходять | `b[1]`, `b[7]` — масивна індексація на MongoDB-об'єктах | Виправлено: `b.telegramId`, `b.time` |
| 3 | ⚠️ Вільні слоти не перевіряються | Показуються всі 9 слотів незалежно від зайнятості | Фільтрувати зайняті слоти з БД |
| 4 | ⚠️ Google Sheets не синхронізується | `booking.js` пише тільки в MongoDB | Додати запис у Sheets після MongoDB |

## 3. Full Bot Logic

### 3.1 Головне меню (/start, /menu)

```
Вітаємо в Romashsq Autoservice!
[📋 Записатись]    [💰 Прайс]
[📍 Адреса]        [🕐 Години роботи]
[📞 Контакт]       [📝 Мої записи]
```

**Правила:**
- `/start` завжди скидає стан і показує меню
- `/menu` — те саме
- Якщо клієнт вже є в БД — персональне привітання ("Вітаємо знову, Іване!")
- Якщо новий — стандартне привітання + пропозиція записатись

### 3.2 Booking Wizard — 7 кроків

```
Крок 1: Вибір послуги
   → 7 кнопок (Механіка, Поліровка, Мийка, Тюнінг, Детейлінг, Шиномонтаж, Діагностика)
   → [❌ Скасувати]

Крок 2: Вибір майстра
   → 2 майстри для обраної послуги
   → [← Назад] [❌ Скасувати]

Крок 3: Вибір дати
   → Наступні 7 робочих днів (Пн–Пт)
   → Формат: "Пн, 24 бер"
   → [← Назад] [❌ Скасувати]

Крок 4: Вибір часу
   → Слоти 09:00–17:00 (9 варіантів)
   → Зайняті слоти НЕ показуються (фільтр по БД: той самий майстер + дата + статус != скасовано)
   → Якщо всі слоти зайняті — повідомлення "На цей день у майстра немає вільних місць" + вибір іншої дати
   → [← Назад] [❌ Скасувати]

Крок 5: Введення телефону
   → Текстове поле: очікується +380XXXXXXXXX
   → Валідація regex: /^(\+380|380|0)\d{9}$/
   → При невірному форматі — повторний запит з прикладом
   → Якщо клієнт вже є в БД і телефон збережений — кнопка "Використати збережений: +380XX..."
   → [❌ Скасувати]

Крок 6: Підтвердження
   → Показ всієї інформації:
      🔧 Послуга: Механіка
      👨‍🔧 Майстер: Іван Петров
      📅 Дата: Пн, 24 бер
      🕐 Час: 10:00
      📞 Телефон: +38 050 123 45 67
   → [✅ Підтвердити] [❌ Скасувати]

Крок 7: Успіх
   → "🎉 Запис підтверджено!"
   → Зберегти в MongoDB (bookings)
   → Зберегти клієнта (upsertClient)
   → Записати в Google Sheets (async, не блокувати відповідь)
   → Надіслати нотифікацію адміну (якщо NOTIFY_ADMIN=true)
   → [🏠 Головне меню]
```

**Важливо:**
- Кнопка "← Назад" має повертати на попередній крок зі збереженим вибором
- `cancel` очищає стан і повертає до меню
- Session зберігається в MongoDB → не залежить від рестартів сервера

### 3.3 Мої записи

```
Список активних записів (статус != скасовано, != завершено)
Для кожного запису:
  📋 Механіка
  👨‍🔧 Іван Петров
  📅 Пн, 24 бер о 10:00
  [❌ Скасувати запис]

Нижче:
  [📋 Новий запис] [🏠 Головне меню]
```

**Логіка скасування:**
1. Натиснути "Скасувати запис"
2. Підтвердження: "Ви впевнені?"
3. [✅ Так] → статус = "скасовано" в MongoDB + в Sheets
4. Повідомлення адміну про скасування

```
[📊 Статистика] [📢 Розсилка]
```

### 4.1 Сьогодні
```
📅 Записи на сьогодні (N):
🕐 10:00 — Механіка
   👤 Іван (+380501234567)
   👨‍🔧 Іван Петров
   [🔧 В роботі] [✅ Готово]
```
- Статуси показуються іконкою: 🕐 підтверджено / 🔧 в роботі / ✅ готово
- При натисканні "✅ Готово":
  1. Статус → "готово" в MongoDB
  2. Записати в History
  3. Надіслати клієнту: "🎉 Ваш автомобіль готовий! Можете забирати."
  4. Записати в Google Sheets History
- [🔄 Оновити] — перезавантажити список

### 4.2 Клієнти
```
Список з пагінацією (10 на сторінку):
👤 Іван Коваленко (@username)
   📞 +380501234567  🚗 Toyota Camry 2019
   Візитів: 5  |  Останній: 20 бер 2026
[← Попередня] [1/3] [Наступна →]
```

## 6. Session Storage (MongoDB)

**Проблема:** `state.js` зберігає стан у `Map` (RAM). При рестарті Render → стан втрачається → "Підтвердити" не працює.

**Рішення:** Session model у MongoDB.

```js
// src/db/models/Session.js
{
  userId: String,   // unique index
  data: Object,     // { mode, step, service, master, date, time, phone }
  updatedAt: Date,  // TTL index: auto-delete after 24h
}
```

```js
// src/state.js — async версія
export async function getState(userId)
export async function setState(userId, data)
export async function clearState(userId)
```

**Всі хендлери** що використовують state стають async (вже є).




## 8. Google Sheets Sync

Sheets є **тільки для читання адміном**. Бот пише в Sheets після кожної успішної операції з MongoDB:

| Операція | MongoDB | Sheets |
|----------|---------|--------|
| Новий запис | bookings.save() | Записи.append() |
| Новий клієнт | clients.upsert() | Клієнти.upsert() |
| Статус "готово" | histories.save() | Історія.append() |
| Скасування | bookings.update() | Записи.updateStatus() |

Sheets запис — **async, не блокує відповідь** боту. Помилка Sheets не впливає на роботу бота.
