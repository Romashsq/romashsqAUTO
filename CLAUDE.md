# Romashsq Autoservice Bot — Project Guide

## Что это

Telegram бот для автосервиса "Romashsq Autoservice" (Одесса, Украина).
Написан на Node.js + grammY. Деплоится на Render.com.

---

## Стек

| Технология | Для чего |
|---|---|
| Node.js + grammY | Telegram бот |
| MongoDB Atlas | Основная БД (клиенты, записи, история) |
| Google Sheets | Дублирование данных для ручного просмотра |
| node-cron | Планировщик напоминаний |
| Render.com | Хостинг |

## Как работает бот (для клиента)

### Главное меню (/start)
```
[📋 Записатись]  [💰 Прайс]
[📍 Адреса]      [🕐 Години роботи]
[📞 Контакт]     [📝 Мої записи]
```

### Флоу записи (7 шагов)
```
1. Выбор услуги (кнопки)
2. Выбор мастера (кнопки)
3. Выбор даты — следующие 7 рабочих дней (кнопки)
4. Выбор времени 09:00–17:00 (кнопки)
5. Ввод телефона +380 (текст, валидация)
6. Подтверждение (кнопки: Підтвердити / Скасувати)
7. Запись сохраняется в MongoDB + Google Sheets
```

### Мои записи
- Показывает активные записи клиента из MongoDB
- Кнопка "Скасувати" → подтверждение → статус меняется на "скасовано"

## Панель администратора (/admin)

Доступна только пользователю с ADMIN_ID.

| Кнопка | Действие |
|---|---|
| 📅 Сьогодні | Все записи на сегодня |
| 🔧 В роботі | Изменить статус → "в роботі" |
| ✅ Готово | Статус "готово" + уведомление клиенту "авто готове" |
| 👥 Клієнти | Список клиентов из MongoDB |
| 📊 Статистика | Кол-во клиентов, записей, выполненных, отмененных |
| 📢 Розсилка | Отправить сообщение всем клиентам |

## Планировщик напоминаний (автоматически)

| Когда | Что делает |
|---|---|
| Каждый день в 10:00 | Напоминание клиентам у кого запись завтра |
| 1-го числа каждого месяца в 12:00 | Пишет клиентам кто не был 6+ месяцев |
| 1 октября и 1 апреля | Сезонная рассылка про шины |

---

## Google Sheets (4 листа)

| Лист | Данные |
|---|---|
| Клієнти | telegram_id, имя, телефон, авто, дата регистрации, визиты |
| Записи | Все записи с статусами |
| Історія | Завершенные работы (используется для 6-месячных напоминаний) |
| Розсилки | Лог отправленных уведомлений (чтобы не дублировать) |

---

## MongoDB (4 коллекции)

| Коллекция | Данные |
|---|---|
| clients | Клиенты (telegramId, имя, телефон, авто, визиты) |
| bookings | Записи (услуга, мастер, дата, время, статус) |
| histories | История выполненных работ |
| notifications | Лог уведомлений для предотвращения дублей |


---

## Деплой на Render

1. Создать репо на GitHub, запушить код (без .env и google-credentials.json)
2. Render → New Web Service → подключить репо
3. Добавить все переменные из .env в Environment Variables на Render
4. Добавить google-credentials.json как отдельную ENV переменную (JSON строкой)
5. WEBHOOK_URL = URL сервиса на Render (например https://romashsq-bot.onrender.com)
